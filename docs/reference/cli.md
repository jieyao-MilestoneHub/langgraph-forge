# CLI Reference

Every `langgraph-forge` subcommand and its flags. Generated by reading
`src/langgraph_forge/scaffold/cli.py`; treat that module as the
authority if anything here looks out of date.

## Synopsis

```
langgraph-forge <command> [flags]
```

Run `langgraph-forge --help` for a top-level overview, or
`langgraph-forge <command> --help` for per-command help.

## Commands

### `init`

Scaffold a new agent project from the template stack.

```
langgraph-forge init <target> \
    --provider <provider> \
    --pattern <pattern> \
    --deploy <deploy> \
    [--mcp <path>] \
    [--no-input] \
    [--force]
```

| Argument / flag | Type | Required | Default | Description |
|---|---|---|---|---|
| `target` | path | yes | — | Directory to create. Last path segment becomes the project name and (after dash → underscore) the inner Python package name. |
| `--provider` | enum | yes | — | LLM provider. One of `anthropic`, `openai`, `grok`, `google`, `bedrock`, `azure`. |
| `--pattern` | enum | yes | — | Agent execution pattern. One of `single`, `supervisor`. |
| `--deploy` | string | yes | — | Deployment adapter. Built-ins: `direct`, `bedrock`, `vertex`, `azure`. Third-party adapters registered via the `langgraph_forge.deployment_adapters` entry-point group also work. Run `list-deploy` to see what's available. |
| `--mcp` | path | no | (none) | Path to an MCP config JSON. Future use; currently only stored in template context. |
| `--no-input` | flag | no | `false` | Disable interactive prompts. Reserved for future interactive fallback when flags are missing; required flags are mandatory regardless. |
| `--force` | flag | no | `false` | Overwrite an existing non-empty target directory. Without this flag, an existing target raises `FileExistsError`. |

**Render order** (later layers override earlier on filename collision):

1. `templates/base/`
2. `templates/patterns/<pattern>/`
3. `templates/providers/<provider>/`
4. The selected adapter's `template_fragment()` directory.

**Generated tree** (for `--provider anthropic --pattern supervisor --deploy direct`):

```
<target>/
├── .env.example
├── .gitignore
├── README.md
├── pyproject.toml
├── src/<package>/
│   ├── __init__.py
│   ├── deploy.py
│   ├── graph.py
│   ├── main.py
│   ├── providers.py
│   └── specialists.py        # supervisor pattern only
└── tests/unit/
    └── test_graph.py
```

### `list-deploy`

Print every registered deployment adapter, one per line, with its
required pip extras.

```
langgraph-forge list-deploy
```

Output format:
```
<name>  extras=<comma-separated extras or '-'>
```

Output (with no third-party adapters installed):
```
azure    extras=azure-ai-agents
bedrock  extras=boto3,bedrock-agentcore
direct   extras=-
vertex   extras=google-cloud-aiplatform
```

Adapters are discovered via the `langgraph_forge.deployment_adapters`
entry-point group; installing a third-party adapter package makes its
name appear here without any change to `langgraph-forge` itself.

### `doctor`

Verify that an adapter's optional extras are installed.

```
langgraph-forge doctor --deploy <name>
```

| Flag | Type | Required | Description |
|---|---|---|---|
| `--deploy` | string | yes | Adapter name (run `list-deploy` to see options). |

**Exit codes:**

- `0` — all required extras importable.
- `1` — adapter unknown, or one or more extras missing. The error
  message includes the exact `pip install langgraph-forge[<name>]`
  command needed.

**Examples:**

```bash
langgraph-forge doctor --deploy direct
# Adapter 'direct' is ready.

langgraph-forge doctor --deploy bedrock      # without bedrock extra
# Adapter 'bedrock' needs: boto3, bedrock-agentcore
# Install with: pip install langgraph-forge[bedrock]
# (exit 1)
```

### `version`

Print the installed `langgraph-forge` version.

```
langgraph-forge version
```

Output:
```
0.1.0a1
```

The value mirrors `langgraph_forge.__version__`, which is sourced from
`src/langgraph_forge/_version.py`.
