# Tutorials

**Audience**: people new to `langgraph-forge` who want to **learn** by
following a step-by-step path from zero to a working agent.

**What goes here**: lessons that take a beginner from one well-defined
starting point to one well-defined ending point with no choices along
the way. The tutorial guarantees that if you follow the steps, you'll
end with the result it promises.

**What does NOT go here**: task-oriented recipes (those go in
[`../how-to/`](../how-to/README.md)), reference material
([`../reference/`](../reference/README.md)), or background explanation
([`../explanation/`](../explanation/README.md)).

## Authoring guidelines

- **One audience**: someone with Python 3.11+ installed and no prior
  exposure to `langgraph-forge`. Don't assume LangGraph familiarity.
- **One result**: every tutorial promises a specific, runnable
  outcome at the end. State it in the first paragraph.
- **No choices**: if there are alternatives, pick one and mention the
  others in a "see also" footer. Tutorials are linear.
- **Verifiable**: every step shows the command and the expected
  output. The reader knows immediately when they've gone off the
  rails.
- **Short**: ~10 minutes of reading + doing. Longer means it's
  probably two tutorials.
